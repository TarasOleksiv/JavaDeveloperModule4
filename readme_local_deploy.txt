Структура проекту:

main/java/...../
    -- controller.servlet
    -- dao
    -- model
    -- CheckDB : клас створення бази з консолі
    -- Settings : клас налаштувань

    -- resources
        -- config.properties - файл конфігурації проекту:
                                     Файл містить параметри шляхів до скриптів ініціалізації та наповнення бази.
        -- DBscheme.png - схема бази.
        -- ehcache.xml - файл налаштування кешу.
        -- hibernate.cfg.xml - файл конфігурації hibernate.
        -- initDB.sql - створює всі необхідні таблиці на базі.
        -- populateDB.sql - заповнює даними таблиці.
        -- log4j.properties - файл конфігурації log4j.

    -- webapp: папка із jsp сторінками
        -- index.jsp - стартова сторінка проекту
        -- manufacturerEdit.jsp - сторінка редагування виробника
        -- manufacturesrsList.jsp - сторінка виводу всіх виробників
        -- productEdit.jsp - сторінка редагування товару
        -- productsList.jsp - сторінка виводу всіх товарів

Схема запуску проекту на локальній машині.

    Етап 0: Підготовка.

    1) Зклонувати проект з гітхабу та відкрити в IDEA
    2) Прописати у файлі resources\hibernate.cfg.xml правильні значення юзера та паролю на свій сервер
    3) Створити war файл проекту засобами Maven: (target/Products.war)
    4) Скопіювати target/Products.war до папки webapps Томкату.
    5) Запустити Томкат (якщо незапущений).
    6) Ініціалізувати базу на сервері mysql.
        Для цього в проекті IDEA запустити на виконання клас CheckDB:
            Програмка перевірить чи база з іменем вказаним у файлі hibernate.cfg.xml існує.
                Якщо база існує, то в консолі виведеться відповідне повідомлення.
                Якщо база не існує, то програмка автоматично запустить на виконання
                    2 скрипти ініціалізації бази та заливки даних (initDB.sql та populateDB.sql з папки resources)
                    і виведе повідомлення в консолі про успішне створення бази.

    Етап 1: Демонстрація роботи web аплікації.

    7) Запустити у броузері стартову сторінку аплікації за лінком
        http://localhost:8080/Products/
    8) На решту сторінок аплікації можна переходити зі стартової сторінки.
